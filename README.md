## ‚ú® What is TaskFlow?

TaskFlow is a **collaborative team task management platform** where teams can create projects, assign tasks, and track progress ‚ÄĒ all with role-based access control. Think of it as a simplified Trello/Asana with a futuristic cyberpunk UI.

---

## ūüöÄ Features

| Feature | Description |
|---|---|
| ūüĒź **JWT Authentication** | Secure signup/login with 7-day token expiry and bcrypt password hashing |
| ūüďĀ **Project Management** | Create projects, invite teammates, manage roles |
| ūüďč **Task Management** | Create tasks with title, description, priority, due date, assignee |
| ūüóāÔłŹ **Kanban Board** | Visual board with To Do / In Progress / Done columns |
| ūüďä **Dashboard** | Real-time stats ‚ÄĒ total tasks, completion rate, overdue, per-member load |
| ūüõ°ÔłŹ **Role-Based Access** | Admins manage everything; Members update only their assigned tasks |
| ūüé® **Futuristic UI** | Cyberpunk dark theme with neon glows, glassmorphism and animations |

## ūüď° API Reference

### Auth
| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/auth/signup` | Register new user |
| POST | `/api/auth/login` | Login ‚ÄĒ returns JWT token |
| GET | `/api/auth/me` | Get current logged-in user |

### Projects
| Method | Endpoint | Access |
|---|---|---|
| GET | `/api/projects` | All members |
| POST | `/api/projects` | Any user |
| GET | `/api/projects/:id` | Members only |
| PUT | `/api/projects/:id` | Admin only |
| DELETE | `/api/projects/:id` | Admin only |
| POST | `/api/projects/:id/members` | Admin only |
| DELETE | `/api/projects/:id/members/:userId` | Admin only |

### Tasks
| Method | Endpoint | Access |
|---|---|---|
| GET | `/api/tasks/dashboard` | All members (stats) |
| GET | `/api/tasks/project/:id` | All members |
| POST | `/api/tasks` | Admin only |
| PUT | `/api/tasks/:id` | Admin (full) / Assignee (status) |
| DELETE | `/api/tasks/:id` | Admin only |

## ‚öôÔłŹ Local Setup

### Prerequisites
- Node.js **v20.x**
- npm v9+
- Git

### 1. Clone the repo
```bash
git clone https://github.com/CoderParva/Task-Flow.git
cd Task-Flow
```

### 2. Install dependencies
```bash
npm run install:all
```

### 3. Create environment files

**`backend/.env`**
```env
PORT=5000
NODE_ENV=development
JWT_SECRET=your-super-secret-key-minimum-32-chars
DB_PATH=./data
FRONTEND_URL=http://localhost:5173
```

**`frontend/.env`**
```env
VITE_API_URL=http://localhost:5000
```

### 4. Run the app
```bash
# Terminal 1 ‚ÄĒ Backend (http://localhost:5000)
npm run dev:backend

# Terminal 2 ‚ÄĒ Frontend (http://localhost:5173)
npm run dev:frontend
```

## ūüĒí Security

- Passwords encrypted with **bcrypt** (10 salt rounds)
- **JWT tokens** verified on every protected route
- **Parameterized SQL queries** ‚ÄĒ no injection possible
- **Server-side role checks** on every mutating endpoint
- Project data isolated ‚ÄĒ users only see their own projects

## ūüß™ How to Test Role-Based Access

1. Sign up as **User A** ‚Üí create a project ‚Üí you are now Admin
2. Open incognito window ‚Üí sign up as **User B**
3. As User A ‚Üí go to Operators tab ‚Üí add User B by email
4. As User A ‚Üí create a task ‚Üí assign it to User B
5. Log in as User B ‚Üí you can only update **status** of your task
6. User B cannot create, edit, or delete anything ‚úÖ
